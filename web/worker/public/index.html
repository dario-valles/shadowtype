What it does

A system-wide ghost-text layer for your whole Mac

It behaves like spell-check: invisible until useful, never in the way. Best-in-class capability — matched locally, on your hardware.

100% on-device & offline

Inference runs locally on Apple Silicon with llama.cpp + Metal. No cloud, no account, no network call for completion — ever. Pull the Wi-Fi and it keeps finishing your sentences. Your keystrokes are read on the device and stay there.

llama.cpp + Metal no inference server no telemetry encrypted profile

Inline ghost-text completion

Faint suggestions appear exactly at the caret in every text field. Tab accepts the next word; ⌥Tab takes the whole line.

Per-app & per-domain control

Enable or silence Shadowtype by app bundle or browser domain. Casual in Slack, formal in Mail — set instruction overrides per app.

Personalization that learns you

An on-device style profile biases completions toward your phrasing — stored encrypted, never transmitted, wipeable anytime.

Autocorrect & typo suppression

A fast heuristic holds back a suggestion when you're mid-typo, and full inline autocorrect fixes transpositions as you go.

Multiple local models

Pick from a catalog of GGUF models — Qwen3 Base (sharp FIM continuation), Gemma 3, MoE and small-MoE variants. Compact, mid-size, and the largest are all included, RAM permitting. Bring your own GGUF too.

Screen-aware context

Optional local OCR reads the focused window so suggestions fit what's on screen, with clipboard awareness for sharper completions.

Emoji & multi-language

Type :smi → 😄. English is sharpest, with working support for Norwegian, Hinglish and more — your language, never sent anywhere.

Selection rewrite

Select any text and press ⌥⌘K to rewrite it inline — improve, shorten, formal, casual, fix grammar, or summarize. Runs on your Mac, previews before it commits. Pro.

Local API + MCP for developers

An OpenAI-compatible HTTP endpoint runs on localhost so your scripts, editors, and agents can call the same on-device model — no key, no cloud. A built-in MCP bridge plugs into Claude Code and other MCP clients. Bring your own GGUF via the BYOM picker.

Questions

Good to know

Does it really work offline?
Yes. All completion happens on your Mac using a local model and Metal — there is no inference network call, ever. The only outbound traffic is the optional update check and the one-time model download. Turn off Wi-Fi and Shadowtype keeps completing.
What about my privacy?
Your keystrokes never leave the device. There is no telemetry, no analytics backend, and no account. Password and secure fields are never read. The personalization profile is stored encrypted on your Mac and you can wipe it any time. You keep full copyright of everything you write.
What does it cost?
Nothing. Shadowtype is free and open source — every feature is on, with no cap, no account, and no subscription. There is no paid tier: system-wide ghost text, Tab-to-accept, emoji, typo suppression, screen-aware context, full autocorrect, personalization, clipboard context, custom & per-app instructions, configurable completion length, the largest models, and Labs features are all included.
Is it really open source?
Yes. The full source is on GitHub. You can read the code, build it yourself, file issues, and contribute. There is no telemetry and no backend to trust — you can verify exactly what runs on your Mac.
Which Macs are supported?
macOS 14 (Sonoma) and later, on Apple Silicon (M1 and up). 8 GB RAM runs the compact + mid models; 16 GB+ unlocks the larger ones. Use it on as many of your Macs as you like.
Can I use my own model, or call it from my code?
Yes. Shadowtype ships a local OpenAI-compatible HTTP API on 127.0.0.1 — your scripts, editor plugins, and agents can hit /v1/chat/completions with no key and no cloud. There's also a built-in MCP server for Claude Code and other MCP clients, and a BYOM picker for any GGUF you drop in. Everything stays on your machine, bound to localhost only.
How do updates work?
Shadowtype checks for new builds in the background and offers a one-click in-place update — signed, notarized, and verified before install. No App Store, no Sparkle, no account. You can also grab any release directly from GitHub.
